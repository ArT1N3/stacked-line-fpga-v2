## 一、总控板 (Master Control Board)

### 1.1 新增功能：高精度可调脉冲方波输出

**需求描述：** 总控板输出一路延时和脉宽均可独立调节的脉冲方波信号。

> **注：脉冲触发通过独立于总分控体系的专用线路分发，不经过光纤环网，不在本 FPGA 工程范围内。**

| 参数 | 规格 |
|------|------|
| 通道数 | 1 路 |
| 输出电平 | LVCMOS 3.3V |
| 延时调节范围 | -1μs ~ +1μs（以触发信号为 0 点） |
| 延时调节步长 | 1ns |
| 脉宽调节范围 | 1ns ~ 300ns |
| 脉宽调节步长 | 1ns |
| 实现方式 | FPGA PLL 倍频 + 进位链精细延时线 |
| 控制接口 | 上位机 UART 命令设置延时值和脉宽值 |
| **GW1N-9C 资源评估** | PLL 可倍频到 ~400MHz (2.5ns粗调) + 进位链(~50ps精调)，约需额外 500 LUT |

## 三、通信系统（光纤手拉手环网）

### 3.1 约束条件

| 约束 | 说明 |
|------|------|
| 环境 | 高电压（450kV）、强电磁干扰 |
| 隔离 | 必须光纤隔离，不使用电信号通信 |
| 拓扑 | 禁止并联（主控不能同时连接所有分控） |
| 光纤数量 | 尽可能少 |
| 光纤总长度 | 尽可能短 |

### 3.2 确定方案：光纤手拉手环网

```
                        ┌─────────────────────────────┐
                        │        光纤1 (下行)           │
                        ▼                             │
  ┌─────────┐    ┌─────────┐    ┌─────────┐       ┌─────────┐
  │ 主控板   │    │ 分控 #1  │    │ 分控 #2  │  ...  │ 分控 #N  │
  │         │    │ RX   TX  │    │ RX   TX  │       │ RX   TX  │
  │  TX  RX │    │  ↓   ↑   │    │  ↓   ↑   │       │  ↓   ↑   │
  └──┬───┬──┘    └──┬───┬──┘    └──┬───┬──┘       └──┬───┬──┘
     │   │          │   │          │   │              │   │
     │   └──────────────┴──────────┴───┴──────────────┘   │
     │           光纤2 (上行)                               │
     └─────────────────────────────────────────────────────┘
```

**只需 2 根光纤**覆盖所有分控，与节点数量无关。

| 特性 | 说明 |
|------|------|
| 拓扑 | 单向环（逻辑环，物理手拉手链） |
| 光纤数量 | **2 根**（下行 + 上行），不随分控数量增加 |
| 光纤总长 | = 相邻节点间距之和，无冗余 |
| 每节点光口 | 1 收 + 1 发（2 个光模块） |
| 电气隔离 | 节点间完全光纤隔离，耐压 >450kV |
| 节点失效处理 | 光旁路开关 / 断电直通 |

### 3.3 数据协议：源路由帧转发

**原理：** 主控发出一个帧，沿下行光纤逐节点传递；目标分控处理后将响应写入帧中继续转发；帧最终从上行光纤回到主控。

```
主控发送帧 ──→ 分控1(不是目标，原样转发) ──→ 分控2(目标匹配，写入响应) ──→ ... ──→ 主控接收
```

**帧格式设计：**

```
┌────────┬────────┬────────┬──────────────┬────────┬────────┐
│ 帧头   │ 目标   │ 命令   │   数据载荷    │  CRC   │ 帧尾   │
│ 1 byte │ 1 byte │ 1 byte │  N bytes     │ 2 byte │ 1 byte │
│ 0xEB   │ 1~127  │        │  (可变长度)   │        │ 0xAA   │
└────────┴────────┴────────┴──────────────┴────────┴────────┘
```

**命令类型：**

| 命令 | 说明 | 数据方向 |
|------|------|----------|
| 0x01 轮询状态 | 查询某分控的故障/继电器状态 | 主控→分控→主控(响应覆写) |
| 0x02 继电器控制 | 控制某分控继电器通/断 | 主控→分控 |
| 0x03 广播 | 同时发给所有分控 | 主控→所有分控 |
| 0xFF 环检测 | 测试环路通断 | 主控→主控(绕环一周) |

**转发规则（每个分控节点）：**
1. 接收帧，检查 CRC
2. 若目标地址 == 本机地址 → 执行命令，将响应写入帧
3. 若 CRC 错误或目标 == 0（广播）→ 也原样转发
4. 立即转发到 TX（直通转发，延迟 < 1μs）

### 3.5 光旁路开关方案对比

光旁路必须在**无电**情况下将光信号从输入光纤直接路由到输出光纤，不能用电气旁路（光模块断电不工作）。

| 方案 | 原理 | 优点 | 缺点 | 单节点成本 |
|------|------|------|------|------------|
| **A. 磁光开关 (推荐)** | 磁保持锁存，上电时切换到"通过光模块"位置；断电时自动弹回直通位置 | 无机械磨损、切换速度快(~1ms)、体积小 | 定制件、需供应商配合 | ~¥60 |
| **B. MEMS 光开关** | 微机电镜片，NC 常闭型 | 成熟商用产品 | 贵、封装大 | ~¥150 |
| **C. 光分路器 + 光开关** | 1×2 分路器分一路常通旁路 | 简单可靠 | 光功率减半，级联后衰减大 | ~¥30 |
| **D. 自保持继电器+光纤跳线** | 机械双刀双掷继电器切换光纤 | 最便宜 | 机械寿命有限、体积大 | ~¥20 |

**推荐方案：机械锁存式 2×2 光开关**

### 3.7 故障响应时序（确认）

```
高压模块短路 → 分控FPGA检测 (本地，<10μs)
                ├─ ① 立即断开继电器 (本地硬件，<1μs)
```

### 3.5 时序分析

100 个分控环网通信性能：

```
帧长 = 10 字节 = 80 bit
光纤速率 = 10 Mbps (FPGA 轻松实现)
每帧传输时间 = 8μs
每节点转发延迟 = ~0.2μs (直通，不缓冲)
环上总延迟 = 100 × 0.2μs + 8μs = 28μs
轮询 100 个分控 = 100 × 28μs ≈ 2.8ms  ← 极快
```

## 五、新增设计要点与注意事项

### 5.1 分控粒度：板级 vs 模块级

分控板控制 1~4 个高压模块，每个模块需要独立的：
- 继电器通断控制
- 短路故障检测

**注意：** 这意味着地址系统需要支持**二级寻址**：
- 环网地址 → 分控板（7bit，最多 127 块板）
- 板内通道 → 高压模块（2bit，1~4 号模块）

帧设计中需包含 `<分控地址> + <模块号>`。

### 5.2 故障响应时序

```
高压模块短路 → 分控检测到故障
                ├─ ① 立即断开对应继电器（本地硬件动作，<1μs）
                └─ ② 等待环网帧经过时，将故障状态写入帧中回传总控
```

**关键要求：** 故障断开必须是**分控本地自主决策**，不能等待总控指令。环网轮询周期可能数毫秒，短路故障不能等。

### 5.3 触发脉冲分发

总控输出 1 路 ns 级可调脉冲。但脉冲模块是**串联**的（Marx 发生器或类似拓扑），各模块的触发有严格的时序关系。

**待确认：** ns 级脉冲是如何分发到几十个串联脉冲模块的？
- [ ] 方案A：通过光纤环网发送触发命令（延时精度受环网转发延时影响，约 ±200ns/节点）
- [ ] 方案B：独立的硬件触发线（同轴电缆手拉手分发，ps 级同步精度）
- [ ] 方案C：总控输出一路光脉冲，经过光纤分束器分发到所有模块（ps 级同步）

> **强烈建议方案 B 或 C。** 光纤环网的通信协议（帧转发）不适合传输 ns 级精度的触发信号。

### 5.4 环网中分控故障的容错

如果某个分控断电或故障，光纤环断裂会导致整个环网通信中断。

**必须实现旁路机制：**
- 分控板断电时，光模块 RX 和 TX 之间通过**旁路继电器（常闭触点）**直连
- 或者使用**光开关**（MEMS optical switch），断电自动直通

### 5.5 地址分配与管理

V1.0 通过拨码开关设置地址（4bit，1~9）。V2.0 需要：

| 参数 | V1.0 | V2.0 |
|------|------|------|
| 分控地址位宽 | 4bit (1~9) | 7bit (1~127) |
| 模块地址 | 无（10通道并行） | 2bit (1~4) 板内选择 |
| 设置方式 | 拨码开关 | 拨码 / 可上位机远程配置 |

**待确认：** 是否支持上位机远程修改分控地址？如支持，需要分控有非易失存储。

### 5.6 通信优先级

环网中可能出现：
- 总控轮询分控状态（常规，低优先级）
- 分控上报故障（紧急，高优先级）

**建议：** 帧格式中增加**优先级标记**，分控在转发帧时可以将故障信息插入正在经过的帧中（无需等待轮询），实现**带内紧急上报**。

### 5.7 总控多任务通信

总控需要同时处理：
1. **上位机通信** — UART，22 字节帧，V1.0 协议沿用
2. **充电机通信** — UART/RS-485，V1.0 协议沿用
3. **分控环网** — 光纤环网，新协议
4. **脉冲输出控制** — 内部高精度时序

**注意：** 总控 FPGA（GW1N-9C）需要同时运行这些逻辑，注意资源评估。如果逻辑资源不够，可能需要升级到 GW2A 系列。
