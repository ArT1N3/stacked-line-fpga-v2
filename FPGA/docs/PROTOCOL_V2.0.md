## 一、物理层

### 1.1 拓扑

```
                          ┌─────────────────────────┐
                          │     光纤1 (下行)          │
                          ▼                         │
  ┌─────────┐       ┌─────────┐       ┌─────────┐   │    ┌─────────┐
  │ 主控板   │       │ 分控 #1  │       │ 分控 #2  │        │ 分控 #N  │
  │         │       │ RX   TX  │       │ RX   TX  │   │    │ RX   TX  │
  │  TX  RX │       │  ↓   ↑   │       │  ↓   ↑   │        │  ↓   ↑   │
  └──┬───┬──┘       └──┬───┬──┘       └──┬───┬──┘   │    └──┬───┬──┘
     │   │              │   │              │   │              │   │
     │   └──────────────────┴──────────────┴───┴──────────────┘   │
     │                 光纤2 (上行)                                 │
     └─────────────────────────────────────────────────────────────┘
```

- **单向逻辑环**：数据流 主控→分控1→分控2→...→分控N→主控
- **2 根物理光纤**：下行 + 上行，不随分控数量增加
- **每节点光口**：1×RX（来自前一节点）+ 1×TX（去往后一节点）

### 1.2 物理参数

| 参数 | 值 |
|------|-----|
| 波特率 | **115200 bps** |
| 数据格式 | 8N1（8数据位，1起始位，1停止位） |
| 空闲电平 | 高 (Mark) |
| 起始位 | 低 (Space) |
| 8×过采样时钟 | 921.6 kHz（50MHz ÷ 52，与 V1.0 相同） |

### 1.3 光纤收发器

| 类型 | 型号 | 波长 | 带宽 |
|------|------|------|------|
| POF（推荐） | AFBR-1521CZ / 2521CZ | 650nm | DC-50MBd |
| 玻璃光纤 | HFBR-1414 / 2412 | 820nm | DC-5MBd |

115200 bps 两种方案均可胜任。

## 二、帧格式

### 2.1 帧结构

```
┌──────┬──────┬──────┬──────┬──────┬──────┬──────────┬──────┬──────┐
│ SOF  │ FLAGS│ DST  │ SRC  │ CMD  │ LEN  │ PAYLOAD  │CRC16 │ EOF  │
│ 1B   │ 1B   │ 1B   │ 1B   │ 1B   │ 1B   │ 0~240B   │ 2B   │ 1B   │
│ 0x7E │      │      │      │      │      │          │      │ 0x7E │
└──────┴──────┴──────┴──────┴──────┴──────┴──────────┴──────┴──────┘
```

| 偏移 | 字段 | 大小 | 说明 |
|------|------|------|------|
| 0 | SOF | 1B | 帧起始定界符 `0x7E` |
| 1 | FLAGS | 1B | 帧控制标志 |
| 2 | DST | 1B | 目标地址 |
| 3 | SRC | 1B | 源地址 |
| 4 | CMD | 1B | 命令码 |
| 5 | LEN | 1B | 载荷长度 (0~240) |
| 6..N | PAYLOAD | 0~240B | 命令载荷 |
| N+1..N+2 | CRC16 | 2B | CRC-16-CCITT，小端序 (LSB 先) |
| N+3 | EOF | 1B | 帧结束定界符 `0x7E` |

- **最小帧** (LEN=0)：9 字节
- **最大帧** (LEN=240)：249 字节
- CRC 覆盖范围：FLAGS、DST、SRC、CMD、LEN、PAYLOAD（不含 SOF/EOF）

### 2.2 FLAGS 字节定义

```
 Bit 7    Bit 6    Bit 5    Bit 4    Bit 3    Bit 2    Bit 1    Bit 0
┌────────┬────────┬────────┬────────┬────────┬────────┬────────┬────────┐
│  RESP  │ BCAST  │  ERR   │  MOD1  │  MOD0  │  RSV   │  RSV   │  RSV   │
└────────┴────────┴────────┴────────┴────────┴────────┴────────┴────────┘
```

| 位 | 名称 | 说明 |
|----|------|------|
| 7 | RESP | 0=请求帧, 1=响应帧 |
| 6 | BCAST | 0=单播, 1=广播 (DST=0xFF) |
| 5 | ERR | 从机置位表示 CRC 错误或处理异常 |
| 4:3 | MOD[1:0] | 模块号: 00=全部, 01=模块1, 10=模块2, 11=模块3/4 |
| 2:0 | RSV | 保留，发送时置 0 |

**模块 4 寻址：** MOD=11 时，若从机配置为 4 模块则寻址模块4，若只有 3 模块则返回错误。

### 2.3 特殊地址

| 地址 | 含义 |
|------|------|
| 0x00 | 主控（响应帧的目标地址） |
| 0x01 ~ 0xFE | 分控地址（最多 254 个） |
| 0xFF | 广播（所有分控处理） |

### 2.4 字节填充 (Byte Stuffing)

确保 SOF/EOF 定界符 `0x7E` 在帧体内唯一出现：

| 原始字节 | 填充后 |
|----------|--------|
| 0x7E | 0x7D 0x5E |
| 0x7D | 0x7D 0x5D |

- **发送端**：SOF 之后、EOF 之前，遇到 0x7E 或 0x7D 进行替换
- **接收端**：SOF 之后遇到 0x7D，将后续字节变换：0x5E→0x7E, 0x5D→0x7D
- CRC 在**原始字节**（去填充后）上计算

### 2.5 CRC-16 规格

| 参数 | 值 |
|------|-----|
| 多项式 | x¹⁶ + x¹² + x⁵ + 1 (0x1021, CRC-16-CCITT) |
| 初始值 | 0xFFFF |
| 输出异或 | 0x0000 |
| 覆盖范围 | FLAGS ~ PAYLOAD 末尾 |
| 传输顺序 | CRC 低字节先 (Little-Endian) |

## 三、命令集

### 3.1 命令总表

| 码 | 名称 | 方向 | 请求载荷 | 响应载荷 | 说明 |
|----|------|------|----------|----------|------|
| 0x01 | STAT_QUERY | M→S→M | 0B | 12B | 单从机状态查询 |
| 0x02 | STAT_COLLECT | M→BC→M | 2B | 4B×N | 广播收集状态（边走边追加） |
| 0x03 | RLY_CTRL | M→S→M | 2B | 1B | 继电器控制 |
| 0x04 | CFG_WRITE | M→S→M | 可变 | 1B | 配置写入 |
| 0x05 | CFG_READ | M→S→M | 1B | 可变 | 配置读取 |
| 0x06 | TOPO_DISCOVER | M→BC→M | 0B | 5B×N | 拓扑发现（物理顺序映射） |
| 0x08 | FAULT_CLEAR | M→S→M | 1B | 1B | 清除故障锁存 |
| 0x09 | RESET_SLAVE | M→S→M | 0B | 0B | 软复位分控 |
| 0x0A | IDENTIFY | M→S→M | 0B | 8B | 读取从机身份 |
| 0x0B | ADDR_ASSIGN | M→S→M | 2B | 1B | 软件分配地址 |
| 0x0D | BROADCAST_RLY | M→BC | 2B | 0B | 广播继电器控制（无响应） |
| 0x0E | BROADCAST_CFG | M→BC | 可变 | 0B | 广播配置（无响应） |

### 3.2 命令详细定义

#### 0x01 — STAT_QUERY（状态查询）

查询指定从机的指定模块状态。

```
请求: CMD=0x01, LEN=0
      FLAGS.MOD = 目标模块 (00=全部模块)

响应: CMD=0x01, LEN=12
      PAYLOAD:
        [0-1]   fault_status    16bit: bit[N]=模块N故障 (N=0..3)
        [2-3]   relay_state     16bit: bit[N]=模块N继电器状态
        [4]     module_present   bit[3:0]=已安装模块位掩码
        [5]     module_count    配置的模块数量 (1~4)
        [6-7]   slave_temp      温度 ADC (16bit有符号, 0.1°C/LSB)
        [8-9]   vcc_monitor     内部电压 (mV)
        [10]    slave_status     bit0=故障锁存, bit1=通信错误, bit2=配置错误
        [11]    reserved         保留
```

#### 0x02 — STAT_COLLECT（广播收集状态）

主控发一帧，所有指定范围内的从机向帧尾追加重载，一圈收齐。

```
请求: CMD=0x02, LEN=2
      FLAGS.BCAST=1, DST=0xFF
      PAYLOAD:
        [0]     group_start     起始地址 (1~254)
        [1]     group_count     收集数量 (1~60)

每从机追加 4B (在范围内才追加):
        [0-1]   fault+relay     bit[0]=故障, bit[1]=继电器, bit[7:4]=模块数
        [2]     address         本机地址
        [3]     status          bit0=故障锁存, bit1=配置错误

响应: CMD=0x02, FLAGS.RESP=1
      PAYLOAD 开头保持 [group_start, group_count]
      后面按物理顺序排列各从机的 4B 追加数据
```

#### 0x03 — RLY_CTRL（继电器控制）

```
请求: CMD=0x03, LEN=2
      FLAGS.MOD = 目标模块 (00=全部)
      PAYLOAD:
        [0]     mask           bit[3:0]=1 表示控制对应继电器
        [1]     value          bit[3:0]=1 闭合, =0 断开

响应: CMD=0x03, LEN=1
      PAYLOAD:
        [0]     ack            0x00=成功, 0x01=无效模块, 0x02=执行失败
```

#### 0x04 — CFG_WRITE（配置写入）

```
请求: CMD=0x04, LEN=2+N
      PAYLOAD:
        [0]     config_id      0x00=模块数量, 0x01=故障阈值, ...
        [1]     data_len       数据长度 N
        [2..]   config_data    配置数据

响应: CMD=0x04, LEN=1
      PAYLOAD:
        [0]     ack            0x00=成功, 0x01=无效ID, 0x02=值超范围
```

#### 0x06 — TOPO_DISCOVER（拓扑发现）

核心用途：获取物理串联顺序与地址编码的映射关系。

```
请求: CMD=0x06, LEN=0
      FLAGS.BCAST=1, DST=0xFF

每从机追加 5B:
        [0-1]   hop_count      从主控开始的跳数 (LE, 0=最近)
        [2]     address        本机地址 (1~254)
        [3]     module_count   配置的模块数 (1~4)
        [4]     status         bit0=故障锁存, bit1=配置错误

响应: CMD=0x06, FLAGS.RESP=1
      主控按 PAYLOAD 中追加顺序（=物理顺序）构建拓扑表
```

**约束：** 每从机 5B，最大载荷 240B → 每帧最多 48 个从机。超过需分组。

#### 0x0A — IDENTIFY（读取身份）

```
请求: CMD=0x0A, LEN=0

响应: CMD=0x0A, LEN=8
      PAYLOAD:
        [0-1]   device_type     0x0001=分控板V2
        [2-3]   fw_version      BCD格式, 如 0x0100=V1.00
        [4-5]   serial_number   生产序列号 (LE)
        [6]     module_count    配置的模块数
        [7]     hw_revision     硬件版本
```

#### 0x0B — ADDR_ASSIGN（地址分配）

```
请求: CMD=0x0B, LEN=2
      PAYLOAD:
        [0]     new_address     新地址 (1~254)
        [1]     write_flash     0x00=仅本次, 0xA5=写入Flash永久保存

响应: CMD=0x0B, LEN=1
      PAYLOAD:
        [0]     ack             0x00=成功, 0x01=地址冲突, 0x02=Flash写入失败
```

#### 0x0D — BROADCAST_RLY（广播继电器控制）

```
请求: CMD=0x0D, FLAGS.BCAST=1, DST=0xFF, LEN=2
      PAYLOAD:
        [0]     mask           bit[3:0]=1 控制对应继电器
        [1]     value          bit[3:0]=1 闭合, =0 断开

响应: 无。所有从机执行但不回写帧。
```

## 四、转发规则

### 4.1 从机状态机

```
                     ┌─────────────────┐
                     │      IDLE        │
                     │  等待 SOF (0x7E)  │
                     └────────┬────────┘
                              │ SOF 检测到
                              ▼
                     ┌─────────────────┐
                     │     RECEIVE      │
                     │  存储字节到缓冲   │
                     │  去填充 0x7D 转义 │
                     │  直到 EOF (0x7E)  │
                     └────────┬────────┘
                              │ EOF 检测到
                              ▼
                     ┌─────────────────┐
                     │     VALIDATE     │
                     │  校验 CRC-16     │
                     └───┬──────┬──────┘
                         │      │
                    CRC OK    CRC FAIL
                         │      │
                         ▼      ▼
                     ┌──────┐ ┌──────────────┐
                     │PROCESS│ │置 ERR=1      │
                     │(4.2)  │ │err_cnt++     │
                     └──┬───┘ │仍转发帧       │
                        │     └──────────────┘
                        ▼
                     ┌─────────────────┐
                     │     FORWARD      │
                     │  字节填充+发送    │
                     │  到下一节点       │
                     └─────────────────┘
```

### 4.2 地址匹配逻辑

```
IF FLAGS.BCAST = 1 AND CMD IN (0x02, 0x06):    -- 收集类广播
    IF address IN [group_start .. group_start+group_count-1]:
        追加本机数据到 PAYLOAD 末尾
        LEN += 追加字节数
        重算 CRC
    转发帧

ELSE IF FLAGS.BCAST = 1:                        -- 非收集类广播
    执行命令 (不写响应)
    转发帧

ELSE IF DST = 本机地址 AND FLAGS.RESP = 0:      -- 单播命中
    执行命令
    将响应写入 PAYLOAD
    更新 LEN
    DST ← SRC (交换地址)
    SRC ← 本机地址
    FLAGS.RESP = 1
    重算 CRC
    转发帧

ELSE IF DST != 本机地址 AND FLAGS.RESP = 1:     -- 响应帧回传
    原样转发 (不修改)

ELSE:                                            -- 不相关帧
    原样转发
```

### 4.3 关键规则

1. **CRC 错误也转发**：不中断环路，仅置 ERR 标志和递增错误计数
2. **FLAGS.RESP=1 时不再修改**：防止响应被后续从机覆盖
3. **收集类命令修改帧后必须重算 CRC**：包括载荷、LEN、CRC 字段全部更新
4. **转发延迟**：存储转发，<帧时间 + ~10个时钟周期 (200ns@50MHz)

## 五、物理顺序映射（拓扑发现）

### 5.1 问题场景

光纤手拉手连接，物理上的串联顺序可能和地址编码完全不同：

```
       光纤入                                    光纤出
         │                                         │
  ┌──────┴──────┐  ┌──────────┐  ┌──────────┐  ┌──┴───────┐
  │  分控 #5    │→│  分控 #1  │→│  分控 #3  │→│  分控 #2  │
  │  (拨码=5)   │  │ (拨码=1)  │  │ (拨码=3)  │  │ (拨码=2)  │
  └─────────────┘  └──────────┘  └──────────┘  └──────────┘

上位机状态栏应显示:  ┌───┬───┬───┬───┐
                    │ 5 │ 1 │ 3 │ 2 │    ← 物理顺序上的地址
                    └───┴───┴───┴───┘
                      位1  位2  位3  位4
```

调试和维护时如果只知道地址不知道物理位置，排查故障极其困难。

### 5.2 拓扑发现流程

```
1. 上位机发出"拓扑发现"指令
2. 主控发送 TOPO_DISCOVER (CMD=0x06, BCAST=1, DST=0xFF)
3. 帧沿光纤环逐节点传递
4. 每个从机在帧尾追加 5B: [跳数2B] [地址1B] [模块数1B] [状态1B]
5. 帧回到主控 → 按追加顺序（=物理顺序）构建拓扑表
6. 主控通过 PC UART 上报拓扑表给上位机
```

### 5.3 拓扑数据上报 PC

主控通过 PC UART 协议（V1.0 22字节帧）上报拓扑表：

```
PC帧: EB + CMD=0x10 + Grp=Rsv + Aux=Rsv + dat1=从机数量 + dat3=拓扑数据 ... + CHK + AA
```

上位机收到后在工作区状态栏按物理顺序显示从机编号及状态：

```
┌──────┬──────┬──────┬──────┬──────┐
│ Pos1 │ Pos2 │ Pos3 │ Pos4 │ ...  │
│ Addr5│ Addr1│ Addr3│ Addr2│      │
│  ✓   │  ✓   │  ✗   │  ✓   │      │
└──────┴──────┴──────┴──────┴──────┘
 正常   正常   故障   正常
```

### 5.4 拓扑变化告警

| 变化 | 告警级别 | 说明 |
|------|----------|------|
| 从机新增 | 提示 | 可能新接入或替换 |
| 从机消失 | **告警** | 断电或光纤断裂 |
| 物理顺序变化 | **告警** | 可能重新接线 |
| 地址冲突 | **严重** | 两个从机同一地址 |

| 策略 | 帧数 | 耗时 | 说明 |

## 七、错误处理

### 7.1 CRC 错误

- 从机检测到 CRC 错误 → 置 FLAGS.ERR=1 → **仍然转发** → 内部错误计数+1
- 主控收到 ERR=1 的帧 → 重发原命令（最多 3 次）
- 连续 CRC 错误超过阈值 → 标记链路质量差

### 7.2 超时重试

```
主控发帧 → 启动超时定时器 (expected_ring_time × 3)
  ├─ 收到响应 → 正常处理
  ├─ 超时 → 重发 (最多 3 次)
  └─ 3次超时 → 标记从机离线
                 ↓
           发送 TOPO_DISCOVER 检查环网状态
```

默认超时时间：`(max_slaves + 1) × max_frame_time × 2 × 3 ≈ 100 × 22ms × 6 ≈ 13.2 秒`

### 7.3 断环检测

1. 主控周期性发送心跳帧 (CMD=0x06, 间隔 500ms)
2. 心跳帧未在超时内返回 → 环断裂
3. 主控逐地址发送 STAT_QUERY (从地址1开始)
4. 最后一个响应的从机 → 断点在其后续链路

### 7.4 错误计数器

每个从机维护（可通过 CFG_READ 读取）：

| 计数器 | 位宽 | 说明 |
|--------|------|------|
| crc_error_cnt | 16bit | CRC 错误次数 |
| frame_lost_cnt | 16bit | SOF无对应EOF次数 (帧碎片) |
| cmd_error_cnt | 16bit | 无效命令次数 |
| forward_cnt | 32bit | 总转发帧数 |

## 八、地址管理

### 8.1 地址来源优先级

```
上电 → 读内部 Flash
         ├─ 有有效地址 → 使用 Flash 地址
         └─ 无效/空白
              ├─ 读拨码开关
              │    ├─ 非 0x00 → 使用拨码地址
              │    └─ 0x00 → 使用默认地址 0x01 + 报告配置错误
```

### 8.2 地址分配流程

1. 出厂：拨码设为 0x00，Flash 空白 → 默认地址 0x01
2. 现场接线：先接光纤环，不管地址
3. 上位机执行"拓扑发现" → 看到所有从机 (默认都显示0x01)
4. 逐从机发送 ADDR_ASSIGN 分配唯一地址 + 写入 Flash (write_flash=0xA5)
5. 再次执行"拓扑发现" → 确认所有地址唯一且物理顺序正确

## 九、与 V1.0 协议对比

| 特性 | V1.0 | V2.0 |
|------|------|------|
| 物理介质 | 同轴电缆 (RXD/TXD) | 光纤 (2根) |
| 协议类型 | UART + 脉宽 双协议 | 统一帧协议 |
| 寻址 | 4bit (1~9) | 8bit 从机 + 2bit 模块 |
| 冲突避免 | 时隙仲裁 (地址×100μs) | 环网转发，无冲突 |
| 多节点上传 | 分时共享 TXD | 帧内逐节点追加 |
| 差错检测 | 13位累加和 | CRC-16-CCITT |
| 帧格式 | 固定 20/22 字节 | 可变长 9~249 字节 |
| 拓扑发现 | 无 | TOPO_DISCOVER 自动发现 |
| 电气隔离 | 无 | 光纤隔离 (450kV耐压) |
| 最大分控 | 12 | 254 |
| 广播 | 无 | BCAST=1 |
| PC 通信 | 22B UART帧 | 不变，完全复用 |
