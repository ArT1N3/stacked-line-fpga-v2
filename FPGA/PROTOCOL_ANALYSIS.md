# 总控 ↔ 分控 通信协议逆向分析

## 一、物理层

```
                        共享单根同轴线 (RXD)
                              │
       ┌──────────────────────┼──────────────────────┐
       │                      │                      │
   ┌───┴───┐             ┌───┴───┐             ┌───┴───┐
   │ 主控板 │             │ 分控#1 │             │ 分控#N │
   │  TXD  │─────────────│  RXD  │─────────────│  RXD  │
   │  RXD  │←────────────│  TXD  │←────────────│  TXD  │
   └───────┘             └───────┘             └───────┘
```

| 参数 | 值 |
|------|-----|
| 物理介质 | 单根同轴电缆（RXD 信号，所有节点共线） |
| 逻辑电平 | RS-232（空闲=高/Mark，起始位=低/Space） |
| TXD 上传 | 分控 TXD → 主控 RXD（同样共享，需仲裁避免冲突） |
| 隔离方式 | 无电气隔离 |

## 二、双协议共存机制

同一根 RXD 线上运行两个完全不同的协议，通过**脉宽长度**区分：

| 特征 | UART 协议 | 脉宽协议 |
|------|-----------|----------|
| 起始位/脉冲宽度 | ~8.7μs (1bit@115200) | **150μs ~ 520μs** |
| 帧长度 | 20 字节 (~1.74ms) | 单个低脉冲 |
| 方向 | 主控→分控（单向） | 主控→分控→主控（查询/响应） |
| 用途 | 配置、复位 | 实时状态查询、充电控制 |
| 冲突避免 | 脉宽检测器阈值 > 150μs，忽略 UART 短脉冲 |

**关键设计：** 分控 `err_order` 脉宽检测器的最小触发阈值约为 150μs（1500×100ns），UART 起始位仅 8.7μs，因此 UART 通信永远不会被误识别为脉宽指令。

## 三、协议一：UART（主控→分控）

### 3.1 物理参数

| 项目 | 值 |
|------|-----|
| 波特率 | 115200 bps |
| 数据位 | 8 |
| 停止位 | 1 |
| 校验位 | 无（应用层校验和） |
| 采样方式 | 8×过采样 (961.5kHz采样时钟) |

### 3.2 帧格式（主控 → 分控：20 字节）

由主控 `mast_to_slave` 模块组包：

```
 ┌──────┬──────┬───────────┬───────────┬─────┬───────────┬──────┬──────┬──────┐
 │ HEAD │ CMD  │ dat1[15:8]│ dat1[7:0] │ ... │dat15[7:0] │CHK_H │CHK_L │ END  │
 │ 0xEB │ 1B   │   1B      │   1B      │     │    1B     │  1B  │  1B  │ 0xAA │
 └──────┴──────┴───────────┴───────────┴─────┴───────────┴──────┴──────┴──────┘
 字节1   字节2   字节3        字节4          字节17      字节18  字节19  字节20
```

**数据段（16 字节 payload）：**
```
CMD + dat1[15:8] + dat1[7:0] + dat3[15:8] + dat3[7:0] + ... + dat15[7:0] = 16 字节
     = 1B命令 + 7×16bit数据字 + 1×8bit数据
```

**校验和：**
- 对字节 2~17（16 字节）求和，取 13 位结果
- 存入 CHK_H（高字节）+ CHK_L（低字节）

### 3.3 命令翻译层

主控在发送前进行命令翻译（`mast_to_slave.cmd_mapped`）：

| PC→主控 CMD | 主控→分控 CMD | 含义 |
|-------------|---------------|------|
| 0x02 | **0x01** (SOH) | 故障复位 |
| 0x05 | **0x55** ('U') | 更新通道配置 |

### 3.4 分控 UART 命令处理

分控 `uart_cmd_deal` 模块处理收到的 UART 帧：

**CMD = 0x55（更新配置）：**
- 锁存 8×16bit = 128bit 配置数据到内部缓冲 `q_buf`
- `q_buf = {dat15, dat13, dat11, dat9, dat7, dat5, dat3, dat1}`
- 根据分控地址 [3:0]，选取对应的 10bit 片段路由到 en_work/en_trig 输出
- 分控地址与数据位映射：

| 地址 | 数据位 | 地址 | 数据位 |
|------|--------|------|--------|
| 0001 | q[9:0] | 0110 | q[59:50] |
| 0010 | q[19:10] | 0111 | q[69:60] |
| 0011 | q[29:20] | 1000 | q[79:70] |
| 0100 | q[39:30] | 1001 | q[89:80] |
| 0101 | q[49:40] | — | — |

- `en_work = 选中片段`（高有效）
- `en_trig = NOT 选中片段`（低有效，适配外部驱动电路）
- `rst_err = '1'`（保持故障检测器活动）

**CMD = 0x01（故障复位）：**
- `rst_err = NOT(confm)` → 产生低脉冲清除所有故障锁存器
- `q_buf` 保持不变（通道配置不丢失）

### 3.5 主控→PC 帧格式（22 字节）

主控通过 `fpga_to_pc` 向上位机回传时使用 22 字节帧，比分控协议多 2 字节（Grp + Aux）：

```
EB + CMD + Grp + Aux + dat1_H + dat1_L + ... + dat15_L + CHK_H + CHK_L + AA
```

## 四、协议二：脉宽协议（主控 ↔ 分控）

### 4.1 工作原理

1. 主控在 RXD 线上产生一个**低电平脉冲**（拉低 → 保持 → 释放）
2. 分控 `err_order` 以 **100ns 分辨率**测量低脉冲持续时间
3. 分控 `err_ctrl` 根据脉宽解码指令类型
4. 分控在 TXD 线上**串行上传 10bit 状态数据**给主控

### 4.2 脉宽测量参数

| 参数 | 值 |
|------|-----|
| 采样时钟 | 10MHz（50MHz÷5，100ns 分辨率） |
| 计数器位宽 | 13bit |
| 最大测量范围 | 8191 × 100ns = 819.1μs |
| 同步器 | 3 级（防止亚稳态） |
| 超时锁存 | 脉宽 ≥ 500μs 自动锁存 |
| 死区 | 锁存后等待 `rst_detector` 释放才能再次测量 |

### 4.3 命令编码

| 脉宽 (实际) | 计数器范围 | 含义 | 分控响应 |
|-------------|-----------|------|----------|
| **150 ~ 220 μs** | 1500 ~ 2200 | 在线状态查询 | 上传电源故障状态 |
| **380 ~ 420 μs** | 3800 ~ 4200 | 使能状态查询 | 上传通道使能状态 |
| **480 ~ 520 μs** | 4800 ~ 5200 | 充电指令 | 使能故障检测 → 等待 posedge → 上传过流状态 |

### 4.4 多节点仲裁（TXD 上传时序）

所有分控共享同一根 TXD 线，通过**地址时隙**避免冲突：

```
分控#1      分控#2      分控#3          分控#12
  │           │           │               │
  ├─ 0μs ─────┤           │               │
  │  上传#1   ├─ 100μs ───┤               │
  │           │  上传#2   ├─ 200μs ───     │
  │           │           │  上传#3        │
  │           │           │        ├─ 1100μs
  │           │           │        │  上传#12
  ▼           ▼           ▼        ▼
  └───────────┴───────────┴────────┘
        每分控 100μs 窗口
```

| 地址 | 延时 | 地址 | 延时 |
|------|------|------|------|
| 0001 | 0 μs | 0111 | 600 μs |
| 0010 | 100 μs | 1000 | 700 μs |
| 0011 | 200 μs | 1001 | 800 μs |
| 0100 | 300 μs | 1010 | 900 μs |
| 0101 | 400 μs | 1011 | 1000 μs |
| 0110 | 500 μs | 1100 | 1100 μs |

### 4.5 TXD 上传帧格式

每个分控上传 **10 位**串行数据：

```
        1位 = 10μs
    ┌────┬────────────┬────┐
    │空闲│  数据 (8μs) │空闲│
    │1μs │            │1μs │
    └────┴────────────┴────┘

  10位 × 10μs = 每分控上传帧占 100μs
```

| 查询类型 | 上传内容 | 编码 |
|----------|----------|------|
| 在线状态查询 | 电源故障状态 err_charge[10:1] | 正常=0, 故障=1 |
| 使能状态查询 | 通道使能状态 en_state[10:1] | 取反输出（线路上低有效） |
| 过流状态查询 | 过流故障 err_total[10:1] | 正常=1, 故障=0 |

**位顺序：** LSB 先出（通道 1→通道 10）

**输出极性：** `mux21a` 对信号取反后输出，配合外部 RS-232 驱动器再次取反，最终得到正确极性。

### 4.6 状态机流程

```
主控发送脉冲        分控端处理
───────────        ──────────

RXD ──┐            FREE（空闲）等待 cmd_flag
       │              │
       ├─ 脉冲宽度 ─→ err_order 测量 (100ns单位)
       │              │
       │           cmd_flag=1 → err_ctrl 解码脉宽
       │              │
       │           ┌──┼──────────────────┐
       │           │  │                  │
       │       on_search           en_search        charge
       │       (在线查询)          (使能查询)       (充电指令)
       │           │                  │                │
       │    等待地址偏移延时      等待地址偏移延时    posedge等待
       │    (addr×100μs)        (addr×100μs)       (RXD上升沿)
       │           │                  │                │
       │      load_1='0'         load_2='0'      load_3='0'
       │           │                  │                │
       │     err_shift_test      err_shift_test   err_shift_test
       │     加载→移位10位       加载→移位10位     加载→移位10位
       │           │                  │                │
       │      shift_done          shift_done       shift_done
       │           │                  │                │
       └───────────┴──────────────────┴────────────────┘
                        ↓
                    FREE（空闲）
```

### 4.7 充电指令特殊流程

充电指令（脉宽 480~520μs）是唯一会**改变分控行为**的脉宽命令：

1. 分控进入 `charge` 状态，`en_charge='1'`（打开故障检测窗口）
2. 分控等待 RXD 上升沿（`posedge`），表示充电脉冲结束
3. 检测到 posedge → `en_charge='0'`（关闭检测窗口）
4. 进入 `charge_search` 状态 → 上传过流检测结果

## 五、主控端命令编码总览

主控 `cmd_deal_m` 模块将上位机发来的 CMD 字节（0x01~0x13）解码为 19 路控制信号：

| CMD | 触发信号 | 用途 | 联动 |
|-----|----------|------|------|
| 0x01 | CMD01 | 保留 | — |
| 0x02 | CMD02 | **故障复位** (→ 分控 0x01) | CMD0A 联锁 |
| 0x03 | CMD03 | 保留 | CMD0A 联锁 |
| 0x04 | **CMD04 (SET_LOCK)** | 参数锁存 | CMD0A 联锁 |
| 0x05 | CMD05 | **使能配置** (→ 分控 0x55) | CMD0A 联锁 |
| 0x06 | CMD06 | 保留 | CMD0A 联锁 |
| 0x07 | CMD07 | 保留 | CMD0A 联锁 |
| 0x08 | **CMD08 (Single)** | 单次触发 | — |
| 0x09 | **CMD09 (BLK_ON)** | 脉冲使能 | — |
| 0x0A | CMD0A | **全局联锁输出** | 任何联锁命令触发 |
| 0x0B | **CMD0B (trig_charge)** | 充电触发 | — |
| 0x0C | **CMD0C (trig_Main)** | 主触发 | CMD0A 联锁 |
| 0x0D | CMD0D | 保留 | CMD0A 联锁 |
| 0x0E | **CMD0E (trig_Ch1)** | 通道1触发 | CMD0A 联锁 |
| 0x0F | **CMD0F (ROM_LK)** | EEPROM 存储 | CMD0A 联锁 |
| 0x10 | CMD10 | 保留 | CMD0A 联锁 |
| 0x11 | **CMD11 (trig_Emg)** | 紧急停止 | CMD0A 联锁 |
| 0x12 | CMD12 | 保留 | CMD0A 联锁 |
| 0x13 | CMD13 | 保留 | CMD0A 联锁 |

**联锁机制：** CMD0A 在几乎所有命令时都会置位（除了 0x01, 0x08, 0x09, 0x0A, 0x0B），作为全局同步/互锁信号。

## 六、协议时序汇总

```
时间轴 (以 100 个分控全轮询为例)
│
├─ UART 配置帧 (CMD=0x55) ───── 1.74ms ─────┤  (主控→所有分控，写配置)
│                                             │
├─ 脉宽 150μs (在线查询)                      │
│   ├─ 分控#1 上传 100μs ─┤                   │
│   ├─ 分控#2 上传 100μs ─┤                   │
│   │       ...                                │  共 ~1.4ms
│   └─ 分控#12 上传 100μs ┤                   │
│                                             │
├─ 脉宽 400μs (使能查询) ── 同上 ~1.4ms ────┤
│                                             │
├─ 脉宽 500μs (充电测试) ── 同上 ~1.4ms ────┤
│
总计：~6ms 完成一次完整巡检 (3 种查询 × 12 分控)
```

## 七、V2.0 迁移注意事项

当前协议设计严重依赖**共享总线**和**时隙仲裁**，直接迁移到光纤环网需要重新设计：

| 当前特性 | V2.0 光纤环网等效 |
|----------|-------------------|
| 脉宽协议（150~520μs 脉冲） | 不再需要——光纤支持全双工帧通信 |
| UART 20 字节帧 | 替换为环网帧格式（目标地址路由） |
| 时隙仲裁（地址×100μs） | 替换为环网逐节点转发（无冲突） |
| 单根 RXD 双协议 | 替换为统一帧协议 |
| 最大 12 分控 | 扩展到 127（7bit 地址） |
| TXD 共享上传 | 替换为上行光纤逐帧回传 |

**建议 V2.0 统一帧格式：**
```
┌──────┬──────┬──────┬──────┬──────────┬──────┬──────┐
│ SOF  │ 目标  │ 源   │ 命令 │ 数据载荷  │ CRC  │ EOF  │
│ 0xEB │ 1B   │ 1B  │ 1B   │  N bytes │ 2B   │ 0xAA │
└──────┴──────┴──────┴──────┴──────────┴──────┴──────┘
```
